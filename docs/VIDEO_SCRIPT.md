# Demo video script — target 2:35

Use only synthetic Maya. Record in English with narration and no background music.

## 0:00–0:18 — Problem

**Screen:** Omyear home.

“A personal birthday book can be meaningful, but making one by hand is slow. Asking a
model to improvise makes it fast but impossible to trust. Omyear keeps the warmth and
shows the work.”

## 0:18–0:42 — Input boundary

**Screen:** `pipeline/examples/maya.json`, briefly show goals/facts.

“We begin with a strict questionnaire: confirmed facts, goals, birth data, the return
location and a gift message. Maya is entirely synthetic. The model cannot add biography

**Screen:** `/pipeline`, scroll through stages.

“Code calculates the symbolic systems and turns every result into a source record.
GPT‑5.6 receives those records and a strict JSON Schema. Every interpretive block must
return exact source IDs. Verification checks citations, exact numbers, all twelve
periods, consecutive dates, locations and language guardrails.”

## 1:18–1:58 — Product experience

**Screen:** `/0811`, show opening, year theme, months, places and practices.

“The verified draft becomes a warm personal book rather than a technical report. It
follows Maya’s year from birthday to birthday, but every chapter remains a reflection
prompt—not an event prediction.”

## 1:58–2:22 — Provenance

**Screen:** provenance section; expand the source ledger.

“This run resolved to GPT‑5.6 Sol. It had 72 source records, cited 53, and finished
with zero unknown citations or structural errors. One paragraph combining three
symbolic frameworks was correctly left for human review.”

## 2:22–2:35 — Codex and close

**Screen:** pipeline hero, then home.

“Codex helped turn an existing handcrafted format into this tested, repeatable engine.
I kept the core decisions: deterministic calculations, synthetic public data and a
human final word. Omyear is a personal year book that can show its work.”

## Recording checklist

- Public YouTube video, under three minutes; spoken audio and reviewed captions.
- No API key, real-person data, photos, private tabs or copyrighted music.
- No third-party logos; show Codex contribution and GPT‑5.6 evidence explicitly.
