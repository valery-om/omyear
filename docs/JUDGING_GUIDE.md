# Omyear judging guide

Thank you for spending a few minutes with Omyear.

## The idea in one sentence

Omyear is a personal interactive book for the year between birthdays—a place to return
to for perspective, encouragement and renewed confidence.

It brings five symbolic self-discovery systems together with a person's real context,
turning fragmented readings into one coherent, source-linked experience. It is a
reflection companion, not a prediction or instruction.

## Fastest path — about three minutes

### 1. Meet the product

Open [app.omyear.com](https://app.omyear.com/?lang=en).

The home page explains the human promise. No account or payment is required.

### 2. Read a complete edition

Open [Maya's Omyear](https://app.omyear.com/0811).

Maya is entirely invented. Browse the portrait, year theme, twelve chapters, places
and practices. Near the end, open **How this book was made** and expand the source
ledger.

### 3. Test the working product

Open [Create yours](https://app.omyear.com/try/?lang=en), choose **Use Maya's sample**
and start generation. English and Russian follow the same pipeline. The result remains
only in the current browser tab.

### 4. Inspect the implementation

Open [How it works](https://app.omyear.com/pipeline) for the seven stages, then review
the [public repository](https://github.com/valery-om/omyear).

## What to notice

- **Complete product experience:** questionnaire, progress, generated result and a
  responsive keepsake—not a prompt demonstration.
- **Bounded GPT‑5.6 synthesis:** the model connects registered inputs and calculations
  into one editorial story; it does not calculate, invent biography or approve itself.
- **Inspectable output:** generated interpretations carry source IDs and exact values
  are restored and checked by code.
- **Human final word:** every run ends at a review boundary, even when automated checks
  pass.
- **Privacy by design:** the public person is synthetic; real questionnaire data and
  generated books are not included in the repository or stored in a database.
