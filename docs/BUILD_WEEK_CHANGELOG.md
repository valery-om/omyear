# Build Week change boundary

Omyear existed before OpenAI Build Week as a handmade gift. The submission does not
present that earlier work as new. It presents the repeatable product built from it
during the submission period.

## Before July 13, 2026

- The founder had created a warm editorial concept and visual language.
- Several private books were assembled by hand for individual recipients.
- An Astro template, person-specific datasets and calculation experiments existed in
  private workspaces.
- There was no public questionnaire-to-book product, reusable input contract,
  source-bounded model workflow or deterministic verification pipeline.

Private recipient data, books, photographs, music and historical Git content are not
included in this repository.

## Built during OpenAI Build Week

1. A generic strict input schema and fully synthetic English example.
2. A generalized deterministic calculation engine and immutable source registry.
3. Source-bounded GPT‑5.6 Structured Output schemas with an exact source-ID enum for
   each parallel segment.
4. An OpenAI Responses API adapter with privacy-preserving safety identifiers.
5. Deterministic checks for citations, exact values, required sections, language and
   candidate locations.
6. A mandatory human-review boundary and compiler into the existing book format.
7. A visible in-book provenance ledger and public seven-stage implementation page.
8. A bilingual, consent-first questionnaire with streamed progress and a private
   browser result.
9. A protected Cloudflare gateway, Vercel Node/Python backend and parallel production
   generation path.
10. Reproducible offline E2E tests, responsive browser QA, security audits and this
    sanitized public repository.
11. A bilingual human-first landing page that separates the emotional promise, five
    symbolic lenses, deterministic code, GPT‑5.6 synthesis and source verification.

## Why the extension is meaningful

Before Build Week, each book required a large amount of manual synthesis and private
person-specific work. The new system accepts a consistent questionnaire, calculates
the source material, uses GPT‑5.6 inside explicit boundaries, verifies the complete
draft and renders a responsive edition through one repeatable sequence.

Codex helped audit the original project, separate private and public material, design
the contracts, implement the pipeline and product flow, test the result and prepare
the judging repository. Founder Valeria Omelnitskaya retained the product concept,
creative direction, safety boundaries and final editorial decisions.
